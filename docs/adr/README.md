# ADR (Architecture Decision Record)

되돌리기 어렵거나 "왜 이렇게 했는지"를 남겨야 하는 기술 결정을 기록한다.
결정 1건 = 파일 1개. 한 번 채택한 ADR은 고쳐쓰지 않고, 바뀌면 새 ADR로 대체한다.

## 언제 쓰나

- 대안이 여럿이고 그중 하나를 골라야 할 때 (동시성 제어, 캐시 전략, 인증 방식 등)
- 나중에 "왜 이 선택?"이 반드시 나올 결정
- 되돌리는 비용이 큰 결정

## 언제 안 쓰나 (정본이 따로 있다)

| 내용 | 정본 |
| --- | --- |
| 코딩 규칙 | `docs/CONVENTIONS.md` |
| 패키지 구조 | `AGENTS.md` |
| API 계약 | `docs/API.md` |
| 데이터 모델 | `docs/ERD.md` |
| 제품 범위 | `docs/PRD.md` |

사소하거나 쉽게 되돌릴 수 있는 결정은 ADR로 남기지 않는다.

## 상태값

| 상태 | 의미 |
| --- | --- |
| 제안됨 | 논의 중. 아직 확정 아님 |
| 채택됨 | 확정. 코드가 이걸 따른다 |
| 폐기됨 | 더 이상 유효하지 않음 |
| 대체됨 | 다른 ADR로 교체됨 (대체한 ADR 번호를 적는다) |

## 파일명

`NNN-제목-kebab.md` (3자리 일련번호). 예: `001-concurrent-order-control.md`
